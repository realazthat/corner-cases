cubelib_corner_t cubelib_get_corner_by_int3(int x, int y, int z)¶

Gets a corner using 3 component integer vector representation.

Each component represents a dimension in the cube; if the component value is more than 0, then it is an upper corner (in that dimension); if it is zero or negative, then it is a lower corner. This accomodates both cube representations, “unit-cube-representation” (where \(x,y,z \in \left\{0,1\right\}\)) or the “origin-centered-cube-representation” (where \(x,y,z \in \left\{-1,+1\right\}\)).

uint_fast8_t cubelib_get_corner_index(cubelib_corner_t corner)¶

Returns a 0-based index that represents this corner; for use in array indexing and similar.

The index returned will be in 3D Z-order curve also known as “Morton order”, within the cube, in the range \([0,7]\).
